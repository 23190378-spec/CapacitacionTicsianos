Modulo Desarrollo de sistemas

🏗️ Submódulo: Sistemas de Información

Este submódulo se centró en la fase de análisis y diseño del ciclo de vida de un sistema. El objetivo fue aprender a traducir las necesidades de un usuario real en diagramas técnicos y estructuras de bases de datos sólidas, asegurando que el sistema sea escalable, lógico y eficiente.Aprendí a visualizar cómo viaja la información de un punto a otro: Diagramas de Flujo de Datos (DFD): Mapeo de las entradas, procesos, almacenamientos y salidas de información. Diccionario de Datos: Definición precisa de cada dato que entra al sistema para evitar ambigüedades. Lógica de Negocio: Establecer las reglas que el sistema debe seguir obligatoriamente (ej. "No se puede vender un producto si el stock es cero").

💻 Submódulo: Programación

Este submódulo se enfocó en el desarrollo del pensamiento lógico-matemático para la resolución de problemas mediante software. El aprendizaje fue progresivo: desde la abstracción visual de un algoritmo hasta la implementación técnica en un lenguaje compilado.Antes de escribir código complejo, aprendimos a pensar como programadores usando pseudocódigo. Estructuras de Control: Uso de condicionales (Si-Entonces) y ciclos (Mientras, Para) para dirigir el flujo del programa. Diagramación: Creación de diagramas de flujo para visualizar la ruta de los datos. Pruebas de Escritorio: Seguimiento manual de las variables para entender cómo cambian los valores paso a paso, lo que ayudó a reducir errores lógicos desde el principio. El uso de un IDE (Entorno de Desarrollo Integrado) fue clave para: Gestión de Proyectos: Organizar múltiples archivos de código. Consola de Comandos: Entender cómo el software interactúa directamente con el sistema operativo a través de una interfaz de texto.

Reflexion

"Programar fue, sin duda, un desafío mental agotador. Al principio, un simple punto y coma (;) olvidado podía causar horas de frustración frente a la pantalla en Dev-C++. Sin embargo, ese mismo estrés fue el que forjó mi paciencia y mi capacidad de análisis. Hoy, al mirar atrás, tengo la experiencia necesaria para entender que programar no es solo escribir comandos, sino resolver acertijos lógicos. Pasar de la duda inicial a ver un código funcionar perfectamente es una de las satisfacciones más grandes que he tenido en mi formación."
